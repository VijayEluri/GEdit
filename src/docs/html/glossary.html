Common Graph Terminology and Definitions
(Outlined in Chapter 12, Sahni, "Data Structures, Algorithms, and Applications in C++")

Graph: A collection of nodes (or vertices) connected by edges.

Node (or vertex): A point in a graph, connected to other nodes by edges.

Edge: A connection between two nodes in a graph. Can be either directed or undirected.

Directed Graph: Graph containing all edges with directed (one way) orientations.

Undirected Graph: Graph containing all edges with no directional orientation.

Adjacent Nodes: Two nodes incident to the same edge.

Weighted Graph: A graph where weights are assigned to each edge. Weighted edges can thus be compared with each other.

Path: A collection of incident nodes and edges, starting at a given node and ending at another node.

Simple Path: A path where all nodes (but start and end nodes) are unique.

Length of Path: Sum of all edges contained in the path.

Subgraph: A subset of nodes and edges of another graph.

Connected Graph: A graph is connected if there is a path between any pair of nodes.

Cycle: A cycle is a path where the starting node is the same as the end node.

Tree: An undirected graph that is connected and doesn't have any cycles.

Spanning Tree: A subgraph tree of a connected graph that connects all nodes in the graph.

Miminum Spanning Tree: A spanning tree on a weighted graph with the smallest possible total edge weight.

Degree (of a Node): The number of undirected nodes incident to a node.

In-Degree (of a Node): The number of directed edges coming into the node.

Out-Degree (of a Node): The number of directed edges leaving a node.

Complete Graph: A graph with an edge between all possible node pairs.

Adjacency Matrix: A matrix used as a method of storing edges between pairs of nodes.

Connected Component: A collection of nodes such that any pair of the nodes is connected by a path.
